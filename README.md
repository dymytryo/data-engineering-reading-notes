# Data Engineering Reading Notes

Personal reading notes for data engineering, data architecture, analytics engineering, infrastructure, and adjacent technical books.

This repository is intentionally organized as a reading and interview-prep reference, not as an installable package or community-maintained project. The notes are meant to capture durable concepts, practical takeaways, and language that can be reused in architecture conversations, interviews, and portfolio storytelling.

## Books

| Book | Focus | Notes |
| --- | --- | --- |
| Fundamentals of Data Engineering | Data engineering lifecycle, architecture, ingestion, transformation, serving, DataOps, governance | [Book notes](books/fundamentals-of-data-engineering/README.md) |

## Note Pattern

Each book should have:

```text
books/<book-slug>/
├── README.md
└── succinct-summary.md
```

Optional files can be added when useful:

```text
interview-prompts.md
chapter-notes.md
architecture-patterns.md
terms.md
```

## Public-Portfolio Guidance

- Keep notes concise and transformative.
- Capture personal takeaways and practical application, not long copied passages.
- Cite the source clearly.
- Avoid uploading full copyrighted books, PDFs, screenshots, or copied chapters.
- Prefer architecture language, trade-offs, and applied examples that show technical judgment.
